# Telegram Analytics Bot

Telegram-бот для анализа сообщений в групповых чатах с использованием OpenAI SDK. Бот собирает сообщения за заданный период, анализирует их с помощью LLM и предоставляет краткую сводку: кто о чем говорил, самые обсуждаемые посты и кто больше всего собрал реакций.

## Возможности

- 📊 Анализ групповых чатов с помощью OpenAI
- 💬 Ответы на вопросы через `/ask` или упоминание `@botname`
- 🖼 Распознавание изображений при ответе на вопросы (vision)
- 💾 Автоматическое сохранение сообщений в SQLite с буферизацией
- ⚡ Кеширование результатов для экономии токенов
- 🛡️ Защита от частых запросов (debounce) для пользователей
- 👥 Пользовательские команды `/anal` и `/ask` для участников группы
- 👨‍💼 Административная панель для управления
- 🐛 Режим отладки для тестирования
- 🔄 Автоматическая очистка старых сообщений
- 🌍 Поддержка часовых поясов для корректного отображения времени
- 📝 Трехуровневая система форматирования (Markdown → HTML → Plain text)
- 🎯 Выбор конкретного чата для анализа (для админа)

## Требования

- Python 3.12+
- Telegram Bot Token
- OpenAI API Key
- Docker и Docker Compose (для запуска в контейнере)

## Установка и запуск

### Вариант 1: Запуск в Docker (рекомендуется)

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram-analytics-bot
```

2. Создайте файл `.env` на основе `.env.example`:
```bash
cp .env.example .env
```

3. Отредактируйте `.env` и заполните необходимые переменные:
```env
# Telegram (обязательные)
BOT_TOKEN=your_bot_token_here
ADMIN_ID=123456789
DEBUG_MODE=false

# OpenAI (обязательные)
OPENAI_API_KEY=your_openai_key_here
OPENAI_BASE_URL=https://api.openai.com/v1
OPENAI_MODEL=gpt-4o-mini
MAX_TOKENS=4000

# Database
DB_PATH=/app/data/bot.db
STORAGE_PERIOD_HOURS=168

# Analysis
ANALYSIS_PERIOD_HOURS=24

# Cache
CACHE_TTL_MINUTES=60

# Debounce (защита от спама)
DEBOUNCE_INTERVAL_SECONDS=21600

# Collection
COLLECTION_ENABLED=true
BUFFER_SIZE=50
BUFFER_FLUSH_INTERVAL_SECONDS=30

# Vision (распознавание изображений)
VISION_MODEL=google/gemini-2.5-flash
VISION_ENABLED=true
VISION_MAX_TOKENS=2000

# Message Formatting
DEFAULT_PARSE_MODE=Markdown
ENABLE_MARKDOWN_ESCAPING=true
MAX_MESSAGE_LENGTH=4096

# Timezone (опционально)
# TIMEZONE=Europe/Moscow
```

4. Соберите и запустите контейнер:
```bash
docker-compose up -d
```

5. Просмотр логов:
```bash
docker-compose logs -f
```

6. Остановка бота:
```bash
docker-compose down
```

### Вариант 2: Локальный запуск с virtualenv

1. Клонируйте репозиторий:
```bash
git clone <repository-url>
cd telegram-analytics-bot
```

2. Создайте виртуальное окружение:
```bash
python -m venv venv
```

3. Активируйте виртуальное окружение:

**Linux/Mac:**
```bash
source venv/bin/activate
```

**Windows:**
```cmd
venv\Scripts\activate
```

4. Установите зависимости:
```bash
pip install -r requirements.txt
```

5. Создайте файл `.env`:
```bash
cp .env.example .env
```

6. Отредактируйте `.env` и заполните необходимые переменные (см. выше)

7. Запустите бота:
```bash
python -m bot.main
```

| `OPENAI_BASE_URL` | Базовый URL для OpenAI API | `https://api.openai.com/v1` |
| `OPENAI_MODEL` | Модель для анализа и ответов | `gpt-4o-mini` |

|------------|----------|--------------|

| `BUFFER_SIZE` | Размер буфера сообщений перед записью в БД | `50` |
| `BUFFER_FLUSH_INTERVAL_SECONDS` | Интервал принудительной записи буфера (секунды) | `30` |

| Переменная | Описание | По умолчанию |

| `ENABLE_MARKDOWN_ESCAPING` | Экранирование спецсимволов Markdown | `true` |

### Распознавание изображений (Vision)

| Переменная | Описание | По умолчанию |
|------------|----------|--------------|

| `VISION_ENABLED` | Включено ли распознавание изображений | `true` |
| `VISION_MAX_TOKENS` | Максимальное количество токенов для ответа vision-модели | `2000` |

### Локализация

| Переменная | Описание | По умолчанию |

## Команды бота

### Пользовательские команды

Доступны всем участникам группового чата:

| Команда | Описание | Период анализа |
|---------|----------|----------------|
| `/anal` | Анализ последних сообщений | 8 часов (настраивается через `ANAL_PERIOD_HOURS`) |
| `/ask <вопрос>` | Задать вопрос боту | Контекст последних 6 часов |
| `@botname <вопрос>` | Задать вопрос через упоминание бота | Контекст последних 6 часов |

**Примечания:**
- Команды работают только в групповых чатах
- Применяется защита от спама (debounce) - повторный запрос возможен через `DEBOUNCE_INTERVAL_SECONDS`
- Для `/ask` и упоминаний интервал настраивается через `INLINE_DEBOUNCE_SECONDS` (по умолчанию 1 час)
- Администратор обходит ограничение debounce
- Результаты кешируются на `CACHE_TTL_MINUTES` минут
- Команда `/ask` и упоминание `@botname` поддерживают цитирование — ответьте на сообщение, чтобы включить его в контекст вопроса
- Ответ на вопрос ограничен 5 предложениями

**Работа с изображениями (Vision):**
- Отправьте фото с подписью `/ask <вопрос>` или `@botname <вопрос>` — бот опишет изображение и учтёт его при ответе
- Ответьте на сообщение с фото командой `/ask <вопрос>` — изображение из цитируемого сообщения будет использовано как контекст
- Отправьте фото без текста с командой `/ask` — бот ответит «Что на этом изображении?»
- Распознавание работает при включённой настройке `VISION_ENABLED` (по умолчанию включено)
- Администратор может переключить распознавание командой `/toggle_vision`

### Административные команды

Доступны только администратору (указанному в `ADMIN_ID`):

| Команда | Где работает | Описание | Пример |
|---------|--------------|----------|--------|
| `/analyze` | Личный чат или группа | Запустить анализ сообщений за период | `/analyze` или `/analyze 12` |
| `/stats` | Личный чат | Получить статистику базы данных | `/stats` |
| `/clear_db` | Личный чат | Очистить всю базу данных | `/clear_db` |
| `/set_storage <hours>` | Личный чат | Установить период хранения сообщений | `/set_storage 168` |
| `/set_analysis <hours>` | Личный чат | Установить период анализа по умолчанию | `/set_analysis 24` |
| `/set_model <model>` | Личный чат | Установить модель OpenAI | `/set_model gpt-4o` |
| `/start_collection` | Личный чат | Возобновить сбор сообщений | `/start_collection` |
| `/stop_collection` | Личный чат | Остановить сбор сообщений | `/stop_collection` |
| `/toggle_vision` | Личный чат | Включить/выключить распознавание изображений | `/toggle_vision` |
| `/manage_groups` | Личный чат | Управление группами (отключение/выход) | `/manage_groups` |

**Особенности команды `/analyze`:**
- В личном чате: показывает список доступных чатов для выбора
- В групповом чате: анализирует текущий чат
- Без параметра: использует период из `ANALYSIS_PERIOD_HOURS`
- С параметром: анализирует указанное количество часов (1-24)
- Администратор обходит debounce и может запускать анализ без ограничений

## Как использовать

### Для администратора

1. Добавьте бота в групповой чат
2. Дайте боту права на чтение сообщений
3. Бот автоматически начнет сохранять сообщения
4. Используйте команды для анализа:
   - В личном чате: `/analyze` - выберите чат из списка
   - В групповом чате: `/analyze` - анализ текущего чата
   - Проверка статистики: `/stats`

### Для участников группы

1. Используйте команды прямо в групповом чате:
   - `/anal` - анализ за 8 часов
   - `/ask <вопрос>` - задать вопрос боту
2. Результаты появятся в том же чате
3. Повторный запрос возможен через 6 часов (настраивается)

## Режим отладки

При включенном `DEBUG_MODE=true`:
- Результаты команды `/analyze` (админ) отправляются только администратору в личные сообщения
- Пользовательская команда `/anal` работает как обычно (в группе)
- Полезно для тестирования без спама в групповом чате

При выключенном `DEBUG_MODE=false`:
- Все результаты анализа отправляются в групповой чат
- Обычный режим работы

**Примечание:** Администратор всегда может запустить `/analyze` в личном чате с ботом для выбора конкретного чата.

## Управление группами

Администратор может управлять группами, в которых работает бот, с помощью команды `/manage_groups`:

### Возможности управления:
- 🟢 **Включить/отключить бота** в конкретной группе
  - Когда бот отключен, он отвечает "Я отключен админом для этой конфы." только на команды и упоминания
  - Обычные сообщения игнорируются и не сохраняются
- 🚪 **Покинуть группу** - бот отправит сообщение "Я ливаю отсюда!" и выйдет из группы, удалив её из базы данных

### Как использовать:
1. Отправьте `/manage_groups` боту в личные сообщения
2. Выберите группу из списка
3. Нажмите нужное действие:
   - "🔴 Отключить" - отключить бота в группе
   - "🟢 Включить" - включить бота обратно
   - "🚪 Покинуть" - бот покинет группу

**Примечание:** Группы автоматически регистрируются в базе данных при добавлении бота или получении первого сообщения.

## Устранение неполадок

### 🔄 Миграция для существующих пользователей

Если вы обновляете бота с предыдущей версии, вам нужно добавить таблицу `groups` в базу данных:

```bash
# Для Docker
docker-compose exec bot python scripts/migrate_add_groups_table.py

# Для локального запуска
python scripts/migrate_add_groups_table.py
```

Миграция автоматически проверит, существует ли таблица, и создаст её при необходимости.

### ⚠️ Сообщения не сохраняются в БД?

**Важно:** Бот обрабатывает только сообщения из групповых чатов!

Если вы тестируете бота в личных сообщениях, сообщения НЕ будут сохраняться.

**Быстрая диагностика:**
```bash
# Проверить состояние БД
python scripts/check_database.py

# Диагностика получения сообщений
python scripts/diagnose_bot.py
```


### Другие проблемы

#### Бот не отвечает на команды
- Проверьте, что `BOT_TOKEN` корректный
- Убедитесь, что бот запущен (`docker-compose logs -f`)
- Проверьте, что ваш Telegram ID совпадает с `ADMIN_ID`

#### Ошибки OpenAI API
- Проверьте корректность `OPENAI_API_KEY`
- Убедитесь, что у вас есть доступ к API и достаточно средств
- Проверьте лимиты токенов (`MAX_TOKENS`)

#### База данных не создается
- Убедитесь, что директория `data/` существует
- Проверьте права доступа к директории
- В Docker убедитесь, что volume правильно примонтирован

#### Debounce блокирует анализ
- Подождите указанное время (`DEBOUNCE_INTERVAL_SECONDS`)
- Или измените интервал в `.env` и перезапустите бота
