# Тестирование функции управления группами

## Подготовка к тестированию

### 1. Миграция базы данных (для существующих установок)
```bash
# Для Docker
docker-compose exec bot python scripts/migrate_add_groups_table.py

# Для локального запуска
python scripts/migrate_add_groups_table.py
```

### 2. Перезапуск бота
```bash
# Для Docker
docker-compose restart bot

# Для локального запуска
# Остановите бота (Ctrl+C) и запустите снова
python -m bot.main
```

## Сценарии тестирования

### Сценарий 1: Автоматическая регистрация группы

**Шаги:**
1. Добавьте бота в новую группу
2. Отправьте `/manage_groups` боту в личные сообщения
3. Проверьте, что группа появилась в списке

**Ожидаемый результат:**
- Группа автоматически зарегистрирована
- Статус: ✅ (включена)
- Отображается название группы

### Сценарий 2: Отключение бота в группе

**Шаги:**
1. Отправьте `/manage_groups` боту в личные сообщения
2. Найдите нужную группу в списке
3. Нажмите кнопку "🔴 Отключить"
4. Отправьте любое сообщение в эту группу

**Ожидаемый результат:**
- Статус группы изменился на ⛔️
- На команды и упоминания бот отвечает: "Я отключен админом для этой конфы."
- Обычные сообщения игнорируются
- Сообщения не сохраняются в базу данных

### Сценарий 3: Включение бота обратно

**Шаги:**
1. Отправьте `/manage_groups` боту в личные сообщения
2. Найдите отключенную группу (⛔️)
3. Нажмите кнопку "🟢 Включить"
4. Отправьте сообщение в группу

**Ожидаемый результат:**
- Статус группы изменился на ✅
- Бот обрабатывает сообщения нормально
- Сообщения сохраняются в базу данных

### Сценарий 4: Выход из группы

**Шаги:**
1. Отправьте `/manage_groups` боту в личные сообщения
2. Найдите нужную группу
3. Нажмите кнопку "🚪 Покинуть"
4. Проверьте группу в Telegram

**Ожидаемый результат:**
- Бот отправляет в группу: "Я ливаю отсюда!"
- Бот покинул группу
- Группа удалена из списка в `/manage_groups`
- Группа удалена из базы данных

### Сценарий 5: Регистрация при первом сообщении

**Шаги:**
1. Добавьте бота в группу, но не отправляйте команды
2. Отправьте `/manage_groups` - группы может не быть в списке
3. Отправьте обычное сообщение в группу
4. Снова отправьте `/manage_groups`

**Ожидаемый результат:**
- После первого сообщения группа появляется в списке
- Статус: ✅ (включена)

### Сценарий 6: Проверка middleware

**Шаги:**
1. Отключите бота в группе через `/manage_groups`
2. Попробуйте выполнить команды в группе:
   - `/anal`
   - `/ask вопрос`
   - `@botname вопрос`
3. Отправьте обычное сообщение без команды

**Ожидаемый результат:**
- На команды и упоминания бот отвечает: "Я отключен админом для этой конфы."
- Обычные сообщения игнорируются (нет ответа)
- Команды не выполняются
- Сообщения не сохраняются

### Сценарий 7: Админские команды в отключенной группе

**Шаги:**
1. Отключите бота в группе
2. Отправьте `/analyze` в этой группе (от админа)

**Ожидаемый результат:**
- Админские команды также блокируются
- Бот отвечает: "Я отключен админом для этой конфы."
- Обычные сообщения от админа игнорируются

### Сценарий 8: Множественные группы

**Шаги:**
1. Добавьте бота в 3-4 группы
2. Отправьте `/manage_groups`
3. Отключите одну группу
4. Проверьте работу в других группах

**Ожидаемый результат:**
- Все группы отображаются в списке
- Отключение одной группы не влияет на другие
- В активных группах бот работает нормально

## Проверка базы данных

### Просмотр таблицы groups
```bash
# Для Docker
docker-compose exec bot python -c "
import asyncio
from database.connection import DatabaseConnection
from config.settings import Config

async def check():
    config = Config.from_env()
    db = DatabaseConnection(config.db_path)
    conn = await db.get_connection()
    cursor = await conn.execute('SELECT * FROM groups')
    rows = await cursor.fetchall()
    for row in rows:
        print(f'Chat ID: {row[0]}, Title: {row[1]}, Enabled: {row[2]}, Added: {row[3]}')
    await db.close()

asyncio.run(check())
"
```

## Логи для отладки

Проверьте логи на наличие записей:
```bash
# Для Docker
docker-compose logs -f bot | grep -i "group"

# Для локального запуска
# Логи выводятся в консоль
```

Ключевые сообщения в логах:
- `"Bot added to group: ..."` - бот добавлен в группу
- `"Group ... registered in database"` - группа зарегистрирована
- `"Bot is disabled for group ..."` - бот отключен в группе
- `"Group ... enabled/disabled"` - изменение статуса
- `"Bot left group ..."` - бот покинул группу

## Откат изменений (если нужно)

Если что-то пошло не так, можно удалить таблицу groups:

```bash
# ВНИМАНИЕ: Это удалит все данные о группах!
python -c "
import asyncio
from database.connection import DatabaseConnection
from config.settings import Config

async def rollback():
    config = Config.from_env()
    db = DatabaseConnection(config.db_path)
    conn = await db.get_connection()
    await conn.execute('DROP TABLE IF EXISTS groups')
    await conn.commit()
    print('Table groups dropped')
    await db.close()

asyncio.run(rollback())
"
```

После этого перезапустите бота - таблица создастся заново.

## Известные ограничения

1. Группы регистрируются только при:
   - Добавлении бота в группу (событие `my_chat_member`)
   - Получении первого сообщения

2. Если бот был добавлен в группу до обновления:
   - Группа появится в списке после первого сообщения
   - Или можно вручную отправить любое сообщение для регистрации

3. Middleware применяется только к обычным сообщениям:
   - Не применяется к системным событиям
   - Не применяется к личным сообщениям боту
