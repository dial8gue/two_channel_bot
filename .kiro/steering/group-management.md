# Управление группами в Telegram Analytics Bot

## Обзор функциональности

Бот поддерживает управление группами через команду `/manage_groups`, которая позволяет:
- Отключать бота в конкретных группах
- Включать бота обратно
- Принудительно покидать группы

## Архитектура

### Модели данных
- `GroupModel` в `database/models.py` - модель группы с полями:
  - `chat_id` - ID чата Telegram
  - `title` - название группы
  - `is_enabled` - статус (включен/отключен)
  - `added_at` - дата добавления

### Репозиторий
- `GroupRepository` в `database/repository.py` - CRUD операции для групп:
  - `add_or_update()` - добавление/обновление группы
  - `get()` - получение группы по ID
  - `get_all()` - получение всех групп
  - `set_enabled()` - включение/отключение группы
  - `delete()` - удаление группы
  - `is_enabled()` - проверка статуса группы

### Сервисный слой
- `AdminService` расширен методами:
  - `get_all_groups()` - получение списка групп
  - `add_or_update_group()` - регистрация группы
  - `toggle_group()` - переключение статуса
  - `is_group_enabled()` - проверка статуса
  - `remove_group()` - удаление группы

### Middleware
- `GroupCheckMiddleware` в `bot/middlewares/group_check_middleware.py`:
  - Проверяет статус группы перед обработкой сообщения
  - Отправляет уведомление "Бот отключен админом для этой группы" если бот отключен
  - Блокирует дальнейшую обработку сообщений из отключенных групп

### Роутеры
- `admin_router.py` расширен:
  - Команда `/manage_groups` - показывает интерактивный список групп
  - Callback handlers для действий: `group_disable`, `group_enable`, `group_leave`
  
- `message_router.py` расширен:
  - Обработчик `on_bot_added_to_group` - автоматическая регистрация при добавлении в группу
  - Автоматическая регистрация группы при получении первого сообщения

## База данных

Таблица `groups`:
```sql
CREATE TABLE groups (
    chat_id INTEGER PRIMARY KEY,
    title TEXT NOT NULL,
    is_enabled INTEGER NOT NULL DEFAULT 1,
    added_at DATETIME NOT NULL
)
```

## Миграция

Для существующих баз данных используйте скрипт:
```bash
python scripts/migrate_add_groups_table.py
```

## Использование

### Для администратора:
1. Отправьте `/manage_groups` боту в личные сообщения
2. Выберите группу из списка
3. Используйте кнопки для управления:
   - "🔴 Отключить" - бот перестанет обрабатывать сообщения
   - "🟢 Включить" - бот возобновит работу
   - "🚪 Покинуть" - бот выйдет из группы

### Автоматическая регистрация:
- Группы регистрируются автоматически при добавлении бота
- Также регистрируются при получении первого сообщения
- По умолчанию все группы включены

## Поведение при отключении

Когда бот отключен в группе:
- На команды (начинающиеся с `/`) отвечает: "Я отключен админом для этой конфы."
- На упоминания (`@botname`) отвечает: "Я отключен админом для этой конфы."
- Обычные сообщения игнорируются (нет ответа)
- Сообщения не сохраняются в базу данных
- Команды не обрабатываются
- Middleware блокирует дальнейшую обработку

## Поведение при выходе из группы

Когда администратор выбирает "🚪 Покинуть":
1. Бот отправляет в группу сообщение: "Я ливаю отсюда!"
2. Ждёт 1 секунду для доставки сообщения
3. Покидает группу через `leave_chat()`
4. Удаляет группу из базы данных

## Технические детали

### Dependency Injection
`GroupRepository` создается в `bot/main.py` и инжектится в `AdminService`:
```python
group_repository = GroupRepository(db_connection)
admin_service = AdminService(
    message_repository=message_repository,
    config_repository=config_repository,
    cache_repository=cache_repository,
    group_repository=group_repository,
    timezone=config.timezone
)
```

### Middleware порядок
1. `CollectionMiddleware` - проверка глобального сбора сообщений
2. `GroupCheckMiddleware` - проверка статуса конкретной группы

Оба middleware применяются только к `message_router`.
